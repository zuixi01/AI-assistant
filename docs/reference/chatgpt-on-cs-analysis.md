## 一、项目概况

| 维度 | 详情 |
|------|------|
| 名称 | 金销数据云智能客服（ChatGPT-On-CS / 懒人客服） |
| 技术栈 | Electron + React + Chakra UI + Zustand + Sequelize (SQLite) + Express |
| 部署形态 | Windows 桌面应用 (.exe)，后转为 SaaS (jinxiaoai.com) |
| 支持平台 | 微信、千牛、拼多多、B站、抖音企业号、抖店、微博、小红书、知乎、京麦、淘宝 |

| 消息类型 | 文本、语音、图片 |

## 二、功能模块拆解

### 2.1 客服工作台

- **多平台统一收件箱**：左侧列表展示所有平台会话，按平台图标区分
- **实时消息流**：WebSocket 推送新消息
- **快捷回复面板**：预设回复模板，分类管理，一键插入
- **人工接管检测**：检测平台侧人工回复后自动暂停 AI
- **聊天记录导出**：Excel 导出，按平台/时间筛选
- **商品卡片关联**：自动关联抖店/千牛商品信息

### 2.2 知识库管理

- **文件上传**：Excel、TXT、Markdown 格式
- **Excel 批量导入导出**：问答对批量管理
- **一键学习真人聊天记录**：从历史对话提取问答对
- **关键词匹配**：精确匹配 + 模糊匹配
- **关键词测试工具**：输入文本预览匹配结果

### 2.3 自动回复策略

- **关键词回复**：精确匹配 → 模板回复
- **AI 智能回复**：LLM 理解意图 → 知识库检索 → 生成回复
- **延时随机回复**：随机 3-8 秒延迟，模拟人工节奏
- **多平台独立配置**：每个平台单独开关
- **回复粒度控制**：全局 / 按平台 / 按会话

### 2.4 数据分析

- **售后咨询分析**：问题类型分布、趋势图
- **意图识别数据**：AI 分类意图的准确率和分布
- **客服绩效**：回复速度、处理量
- **平台对比**：各平台消息量、回复率对比

### 2.5 商品同步

- **一键商品同步**：从抖店/千牛拉取商品列表
- **商品知识库**：商品信息自动转化为知识库条目

## 四、不建议复用模块

| 模块 | 原因 |
|------|------|
| Electron 桌面架构 | 当前项目是服务端 SaaS |
| Sequelize/SQLite | 当前使用 Prisma + PostgreSQL |
| 浏览器自动化适配 | 当前通过聚光 API/Webhook |
| Chakra UI | 当前使用 Tailwind CSS |
| Zustand | 当前使用 React hooks |
| AGPL 许可证代码 | 任何引用都可能触发许可证传染 |

## 五、AGPL-3.0 许可证风险说明

### 风险等级：高

**核心约束：**
1. 网络交互传染：通过网络与 AGPL 软件交互的用户必须获得源代码访问权
2. 修改传染：对 AGPL 代码的修改必须以相同许可证开源
3. 链接传染：动态链接也被视为衍生作品

**安全做法：**
- ✅ 仅阅读源码获取产品灵感
- ✅ 参考 API 设计思路，自行实现
- ✅ 以 sidecar 模式独立运行（不修改源码）
- ❌ 直接复制代码片段
- ❌ 将 AGPL 代码作为库引入
- ❌ 修改后作为闭源产品使用
